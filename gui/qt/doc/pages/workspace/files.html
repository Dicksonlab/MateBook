Files Tab

The "Files" tab views all files within a project that were added by the user. Each row in this view represents one file and the columns contain information for each file. Depdending on the represented type of file, some of the columns may remain empty. Each file is within a certain stage having a certain status. There are six stages a file can be in:

  • Recording: When adding a file to a project and no further operations were conducted, the file is in this stage.
  • Arena Detection: Describes the process of detecting vessels, where the flies are held, in video files.
  • Fly Tracking: Describes the process of analysing an detected arena for courtshipbehavior of two flies in video files.
  • Pulse Detection: Describes the process of detecting pulses of a courtshipsong in audio files.
  • Modifying File: The user is able to modify an audio file after the pulse detection was performed.
and each stage has one of four available status: Failed, Queued, Started, Finished.





Also the user can modify certain cells by double clicking on them. Cells that can be modified are:

  • Path: The user can select the new file path by choosing the new file location using a file dialog.
  • Noise File: To be able to perform a pulse detection on an audio file, a file containing the background noise is needed. If this column is empty the default noise file, provided by the program, is used.
  • Experimenter: coming soon ...
  • Date: By default this column contains the creation date of a file.
  • # sex: The user can define the sex of a fly.
  • # genotype: Comment about the genotype of a fly.
  • Arena size: coming soon ...
  • Start: coming soon ...
  • End: coming soon ...
  • Comment: Containing comments regarding the selected file.
