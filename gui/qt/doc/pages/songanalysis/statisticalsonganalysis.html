Statistical Song Analysis

The statistical song analysis can be devided into two steps. First the song is searched for pulses, which pass the checks using userdefined statistical options. Secondly statistical figures are calculated, using the pulse data that is left. Statistical figures which are calculated are:

  • mean interpulse interval
  • mean cycles per pulse
  • mean train length (defined by the number of pulses inside a train
  • standard deviation of each mean
  • standard error of each standard deviation

Further when calculating statistical figures for more than one file, the mean mean, mean standard deviation and mean standard error are calculated using the statistical figures that were calculated before for each individual file.

Example: The mean mean of 3 files is calculated by adding up the 3 mean values mean1, mean2, mean3 and devide them by 3. Like this each file should have equal impact. A file containing only 3 pulses has the same power as a file containing 100 pulses.


Statistical Options: The user can set the parameters the statistical song analysis should run with, by choosing "Settings" in the menubar --> "Song Analysis" --> "Song Statistics Options". Following parameters are available:

    IPI Options:

  • Min. train length (pulses): Defines how many pulses a train must have, so that pulses inside this train are taken into account for statistical IPI calculations.
  • Minimal IPI distance (ms): The minimum distance in milliseconds two neighboring pulses must have to be taken into account for the whole statistical analysis (IPIs, trains, cycles per pulse, cycles per minute).
  • Maximal IPI distance (ms): The maximum distance in milliseconds two neighboring pulses must have to be taken into account for the whole statistical analysis (IPIs, trains, cycles per pulse, cycles per minute).
  • IPI bin size (ms): Defines the range of one bin in milliseconds. (For graphical representation of calculated statistical IPI data)

    Train Options

  • Min. train length (pulses): Defines how many pulses a train must have, so that pulses inside this train are taken into account for statistical train calculations.
  • Train bin size (pulses): Defines the pulse range number inside one bin. (For graphical representation of calculated statistical train data)
  • Pulse/min excluding pulses in groups of:Defines how many pulses must be in a group to be taken into account for calculation of the pulses per minute.

    Cycle Options

  • Min. train length (pulses): Defining how many pulses a train must have, so that pulses inside the train are taken into account for statistical cycles per pulse calculations.
  • Min. number of cycles (#): Defining how many cycles a pulse must have to be taken into account for statistical cycles per pulse calculations.

    Batchrun Options

  • Min. number of pulses (#): The minimum number of pulses a song must have, to be taken into account for the batchrun calculations. This option concerns the mean, standard deviation and standard error for cycles per pulse and pulses per minute.
  • Min. number of IPIs (#): The minimum number of IPIs a song must have, to be taken into account for the batchrun calculations. This option concerns the mean, standard deviation and standard error for IPIs.
  • Min. number of trains (#): The minimum number of trains a song must have, to be taken into account for the batchrun calculations. This option concerns the mean, standard deviation and standard error for trains.

The number of pulses that are used for calculating statistical figures on IPIs, trains and cycles per pulse are determined seperately. Only the minimal and maximal IPI distances are used to determine all real pulses, which form the basis for all further calculations.

Cycles per pulse count: After the pulses of a song were detected, it is necessary to determine the cycles per pulse (cpp) count. To get this number all the negative peaks and all the positive peaks of a pulse are counted. The smaller one of this two sums is taken as cpp count. A Peak starts at height zero and ends at height zero. The heighest value between those two points represents the peak height.
